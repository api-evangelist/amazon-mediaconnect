<h2>Introduction</h2> 
<p>In this post, we focus on live content generated in real time, as well as file playout or VOD-to-live workflows where the assets are prepared ahead of time. <a href="https://aws.amazon.com/elemental-live/">AWS Elemental Live</a> (on-premises) and <a href="https://aws.amazon.com/medialive/">AWS Elemental MediaLive</a> (cloud) both support live HDR workflows up to UHD, as well as the ability to down-convert HDR to SDR. These platforms also allow forcing a mix of HDR formats in the input side to a single output type, which can be beneficial downstream.</p> 
<h2>Workflows</h2> 
<p>For HDR cloud-based workflows, AWS Elemental MediaLive can receive live sources via HLS, <a href="https://aws.amazon.com/mediaconnect/">AWS Elemental MediaConnect</a> flows, uncompressed <a href="https://aws.amazon.com/media-services/resources/cdi/">AWS Cloud Digital Interface (AWS CDI)</a> sources, or inputs from <a href="https://aws.amazon.com/medialive/features/link/">AWS Elemental Link</a>, which supports 10-bit video. MP4 assets that live in <a href="https://aws.amazon.com/s3/">Amazon Simple Storage Service (Amazon S3</a>) can loop or play out as part of a VOD-to-live workflow.</p> 
<p>For HDR on-premises workflows, AWS Elemental Live can receive content via unicast, multicast, SDI, SMPTE ST 2110, SMPTE 2022-6, HLS, SRT, or directly from MediaConnect. Media files upload directly to AWS Elemental Live and play out as inputs for your live event. Optionally, external devices such as an <a href="https://www.aja.com/products/ki-pro-ultra-12g">AJA Ki Pro Ultra</a> can handle file playout and feed an AWS Elemental Live appliance via 3G SDI or quad-SDI for UHD sources.</p> 
<p>Depending on the scale of your production, live streaming in HDR can present obstacles to overcome in order to keep your color choices accurate. Some cameras, such as many made by Blackmagic Designs, allow you to shoot in logarithmic format and apply a look up table (LUT) in the camera directly, so the output is already PQ or HLG. In some cases, where loading a LUT into the camera is not possible, customers can handle the log to PQ/HLG/SDR conversion using something like an <a href="https://www.aja.com/products/fs-hdr">AJA FS-HDR</a>, which is also a great resource to feed your on-set monitors and ensure the look is exactly how the producers envisioned it.</p>

<h2>Metadata</h2> 
<p>If your outputs are HDR10 or Dolby Vision compliant, some analysis is required to extract the MaxCLL and MaxFALL values needed. If you play out a file as your source, you can use a tool like DaVinci Resolve to export the HDR10 metadata when it completes an HDR10+ inspection of each frame. You can review the text file to find the needed values.</p> 
<p>If you’re in a live streaming scenario, this becomes more of a best effort. Depending on your event, you can record the rehearsal, or record the sport field before the event begins and run through the same metadata extraction process.</p> 
<p><img alt=" &nbsp;The MaxCLL and MaxFALL extracted by DaVinci Resolve" class="aligncenter wp-image-9535 size-full" height="264" src="https://d2908q01vomqb2.cloudfront.net/fb644351560d8296fe6da332236b1f8d61b2828a/2021/09/13/ThSept16_20212.png" width="977" /></p> 
<p>When targeting a Dolby Vision output, AWS Elemental Live is required for the workflow. Since AWS Elemental Live has integrated the Dolby Vision SDK, the software adds the necessary metadata in the outputs where Dolby Vision Profile 5 or 8.1 is selected in the <strong>Preprocessors &gt;&gt; Color Corrector &gt;&gt; Color Space Conversion</strong> section of the output. Read this <a href="https://aws.amazon.com/blogs/media/getting-started-with-aws-elemental-live-and-dolby-vision/">deep dive</a> on Dolby Vision for more information.</p>

<h2>HEVC &amp; AVC</h2> 
<p>When dealing with ABR protocols like HLS or DASH, expecting the client player to switch between codecs is not recommended. Many players, such as smart televisions or older USB stick devices, don’t have the CPU necessary to switch between codecs and will buffer or stall completely. Players also shouldn’t switch between HDR and SDR. This can cause a wrinkle when dealing with HDR, as it means you likely need a full HDR ABR ladder, a full SDR ABR ladder, and if you want to support both HEVC and AVC, you likely need a third ladder. This may require using multiple AWS Elemental Lives for the same stack, which requires the Output Locking feature to synchronize outputs at the frame level. Read this <a href="https://aws.amazon.com/blogs/media/output-locking-in-aws-elemental-live/">walkthrough</a> of Output Locking for more information. Additionally, the mix of HDR and SDR can cause challenges when dealing with SSAI, where the ads may be in SDR, but the program is in HDR. It’s critical to test your players with all the expected variations to ensure smooth playback for your users. Many TVs will allow USB sticks to play back video content, making it an easy way to double check your HDR settings on a TV.</p> 
<h2>HDR product support reference</h2> 
<p><em>AWS Elemental Live:</em><br /> Inputs: HLG, HDR10(PQ)<br /> Outputs: HLG, HDR10(PQ) and Dolby Vision</p> 
<p><em>AWS Elemental MediaLive</em><br /> Inputs: HLG and HDR10(PQ)<br /> Outputs: HLG(Passthrough), HDR10(PQ)<br /> The simulated difference between HDR and SDR</p>

<h2>Conclusion</h2> 
<p>In this three part series, we walked through the AWS appliances and services used to add HDR support to your OTT video workflows. Adding HDR can make a major impact to your viewers and elevate your content, but it does require some upfront planning to ensure the color makes it to the screen exactly as it was intended. 10-bit video is key and having both an HDR and SDR monitor helps guarantee an excellent end result.</p>
